# CRMLite

Django REST API CRM система для управления компаниями, пользователями, складами, поставщиками, товарами и продажами с JWT аутентификацией.

## Возможности

- **Пользователи**: регистрация, аутентификация, управление профилем
- **Компании**: создание, редактирование, удаление (ИНН, владелец)
- **Сотрудники**: прикрепление/удаление пользователей к компании
- **Склады**: управление хранилищами с привязкой к компаниям
- **Поставщики**: CRUD операции для контрагентов
- **Товары**: управление товарами с ценами закупки и продажи
- **Поставки**: пополнение остатков товаров на складе
- **Продажи**: оформление продаж с автоматическим списанием остатков
- **Безопасность**: JWT токены (60 мин), права доступа, контроль остатков

## API эндпоинты

### Аутентификация
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/register/` | Регистрация пользователя |
| POST | `/api/login/` | Вход в систему |

### Компании
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/company/create/` | Создание компании |
| GET | `/api/company/` | Получение компании |
| PUT/PATCH | `/api/company/update/` | Обновление компании |
| DELETE | `/api/company/delete/` | Удаление компании |

### Сотрудники
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/company/attach-user/` | Прикрепление сотрудника |
| GET | `/api/company/employees/` | Список сотрудников |
| DELETE | `/api/company/employees/<id>/` | Удаление сотрудника |

### Склады
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| POST | `/api/storage/create/` | Создание склада |
| GET | `/api/storage/` | Получение склада |
| GET | `/api/storage/<id>/` | Получение склада по ID |
| PUT/PATCH | `/api/storage/update/<id>/` | Обновление склада |
| DELETE | `/api/storage/delete/<id>/` | Удаление склада |

### Поставщики
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/suppliers/` | Список поставщиков |
| POST | `/api/suppliers/` | Создание поставщика |
| GET | `/api/suppliers/<id>/` | Получение поставщика |
| PUT/PATCH | `/api/suppliers/<id>/` | Обновление поставщика |
| DELETE | `/api/suppliers/<id>/` | Удаление поставщика |

### Товары
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/products/` | Список товаров |
| POST | `/api/products/` | Создание товара (quantity=0) |
| GET | `/api/products/<id>/` | Получение товара |
| PUT/PATCH | `/api/products/<id>/` | Обновление товара |
| DELETE | `/api/products/<id>/` | Удаление товара |

### Поставки
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/supplies/` | Список поставок |
| POST | `/api/supplies/create/` | Создание поставки |

### Продажи
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/sales/` | Список продаж (пагинация, фильтры) |
| POST | `/api/sales/create/` | Создание продажи |
| GET | `/api/sales/<id>/` | Получение продажи |
| PUT/PATCH | `/api/sales/<id>/` | Обновление продажи |
| DELETE | `/api/sales/<id>/` | Удаление (возврат на склад) |

## Технологии

- **Backend**: Django 6.0.3
- **API Framework**: Django REST Framework 3.17.0
- **Аутентификация**: Simple JWT 5.5.1
- **Документация API**: drf-spectacular 0.29.0
- **База данных**: SQLite (dev) / PostgreSQL (prod)
