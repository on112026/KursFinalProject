## 1. Доступ к админ-панели

### URL
```
http://127.0.0.1:8000/admin/
```

### Требования к пользователю

Для входа в админ-панель пользователь должен:
- Иметь аккаунт в системе (email и пароль)
- Быть сотрудником компании (`is_staff=True`)

### Вход в систему

1. Откройте браузер и перейдите по адресу `http://127.0.0.1:8000/admin/`
2. Введите email в поле **Username**
3. Введите пароль в поле **Password**
4. Нажмите кнопку **Log in**

**Пример данных для тестирования:**
- Email: `owner@example.com`
- Пароль: `ownerpass123`

### Выход из системы

Для выхода нажмите кнопку **Log out** в правом верхнем углу или перейдите по адресу:
```
POST http://127.0.0.1:8000/admin/logout/
```

### Защита страниц

При попытке доступа к `/admin/` без авторизации система автоматически перенаправит на страницу входа:
```
/admin/login/?next=/admin/
```

## 2. Управление компаниями (Company)

Компания — основная сущность системы. Каждый пользователь может быть владельцем одной компании.

### URL для управления
```
/admin/api/company/
```

### Структура модели Company

| Поле | Тип | Описание |
|------|-----|----------|
| inn | CharField (12 символов) | Уникальный идентификатор (ИНН) |
| name | CharField (255 символов) | Название компании |
| owner | ForeignKey (User) | Владелец компании |
| users | ManyToMany (User) | Сотрудники компании |

### Создание компании

1. Перейдите в раздел **COMPANY**
2. Нажмите кнопку **Add Company**
3. Заполните поля:
   - **INN** — 12-значный уникальный номер
   - **Name** — название компании
   - **Owner** — выберите пользователя-владельца
4. Нажмите **Save**

**Важно:**
- Один пользователь может быть владельцем только одной компании
- ИНН должен быть уникальным
- Владелец автоматически получает доступ к управлению компанией

### Просмотр списка компаний

1. Перейдите в раздел **COMPANY**
2. Отобразится список всех компаний, к которым у вас есть доступ

### Редактирование компании

1. В списке компаний нажмите на название компании
2. Внесите необходимые изменения
3. Нажмите **Save**

### Удаление компании

1. В списке компаний выберите нужную
2. Нажмите ссылку **Delete**
3. Подтвердите удаление

**Примечание:** Удаление доступно только суперпользователям или пользователям с соответствующими правами.

## 3. Управление складами (Storage)

Склад принадлежит одной компании и хранит информацию о товарах.

### URL для управления
```
/admin/api/storage/
```

### Структура модели Storage

| Поле | Тип | Описание |
|------|-----|----------|
| address | TextField | Адрес склада |
| company | OneToOneField (Company) | Компания-владелец |

### Создание склада

1. Перейдите в раздел **STORAGE**
2. Нажмите **Add Storage**
3. Заполните поля:
   - **Address** — полный адрес склада
   - **Company** — выберите компанию
4. Нажмите **Save**

### Особенности

- Каждая компания может иметь только один склад (OneToOne)
- Склад автоматически связан с товарами этой компании

## 4. Управление поставщиками (Supplier)

Поставщик — компания или лицо, поставляющее товары.

### URL для управления
```
/admin/api/supplier/
```

### Структура модели Supplier

| Поле | Тип | Описание |
|------|-----|----------|
| name | CharField (255 символов) | Название поставщика |
| inn | CharField (12 символов) | ИНН поставщика |
| contact_info | TextField | Контактная информация |
| company | ForeignKey (Company) | Компания-заказчик |

### Создание поставщика

1. Перейдите в раздел **SUPPLIER**
2. Нажмите **Add Supplier**
3. Заполните поля:
   - **Name** — наименование поставщика
   - **INN** — идентификационный номер (необязательно)
   - **Contact info** — телефон, email и т.д.
   - **Company** — ваша компания
4. Нажмите **Save**

### Редактирование и удаление

- Для редактирования нажмите на название поставщика
- Для удаления используйте ссылку **Delete**

## 5. Управление товарами (Product)

Товар хранится на складе и имеет информацию о ценах и количестве.

### URL для управления
```
/admin/api/product/
```

### Структура модели Product

| Поле | Тип | Описание |
|------|-----|----------|
| title | CharField (255 символов) | Название товара |
| description | TextField | Описание товара |
| purchase_price | DecimalField | Цена закупки |
| sale_price | DecimalField | Цена продажи |
| quantity | PositiveIntegerField | Количество на складе |
| storage | ForeignKey (Storage) | Склад хранения |

### Создание товара

1. Перейдите в раздел **PRODUCT**
2. Нажмите **Add Product**
3. Заполните поля:
   - **Title** — название товара
   - **Description** — описание (необязательно)
   - **Purchase price** — цена закупки (например, 100.00)
   - **Sale price** — цена продажи (например, 150.00)
   - **Quantity** — начальное количество (может быть 0)
   - **Storage** — выберите склад
4. Нажмите **Save**

### Просмотр товаров

В списке товаров отображаются:
- Название товара
- Закупочная и продажная цены
- Количество на складе
- Склад хранения

### Редактирование товара

1. Нажмите на название товара
2. Измените необходимые поля
3. Нажмите **Save**

**Примечание:** Изменение quantity вручную возможно через админку. Автоматическое обновление остатков происходит только через API при создании поставок и продаж.

## 6. Управление поставками (Supply)

Поставка — это операция закупки товаров у поставщика.

### URL для управления
```
/admin/api/supply/
```

### Структура модели Supply

| Поле | Тип | Описание |
|------|-----|----------|
| supplier | ForeignKey (Supplier) | Поставщик |
| storage | ForeignKey (Storage) | Склад получателя |
| delivery_date | DateTimeField | Дата доставки (автоматически) |

### Создание поставки (пошагово)

#### Шаг 1: Создание записи поставки

1. Перейдите в раздел **SUPPLY**
2. Нажмите **Add Supply**
3. Заполните поля:
   - **Supplier** — выберите поставщика
   - **Storage** — выберите склад
4. Нажмите **Save**

#### Шаг 2: Добавление товаров к поставке

1. Перейдите в раздел **SUPPLYPRODUCT**
2. Нажмите **Add Supply product**
3. Заполните поля:
   - **Supply** — выберите созданную поставку
   - **Product** — выберите товар
   - **Quantity** — количество единиц
4. Нажмите **Save**

### Пример создания поставки

```
Поставщик: ООО "Рога и копыта"
Склад: ул. Пушкина 10, Москва
Товар: Ноутбук ASUS ROG (10 шт.)
```

## 7. Управление продажами (Sale)

Продажа — операция реализации товаров покупателю.

### URL для управления
```
/admin/api/sale/
```

### Структура модели Sale

| Поле | Тип | Описание |
|------|-----|----------|
| company | ForeignKey (Company) | Компания-продавец |
| buyer_name | CharField (255 символов) | Имя/название покупателя |
| sale_date | DateTimeField | Дата продажи |
| total_amount | DecimalField | Общая сумма |

### Создание продажи (пошагово)

#### Шаг 1: Создание записи о продаже

1. Перейдите в раздел **SALE**
2. Нажмите **Add Sale**
3. Заполните поля:
   - **Buyer name** — имя или название покупателя
   - **Company** — ваша компания
   - **Sale date** — дата продажи (ISO формат)
4. Нажмите **Save**

#### Шаг 2: Добавление товаров к продаже

1. Перейдите в раздел **PRODUCTSALE**
2. Нажмите **Add Product sale**
3. Заполните поля:
   - **Sale** — выберите созданную продажу
   - **Product** — выберите товар
   - **Quantity** — количество единиц
   - **Price** — цена за единицу
4. Нажмите **Save**

### Пример создания продажи

```
Покупатель: Иванов Иван Иванович
Компания: Тестовая Компания ООО
Товар: Ноутбук ASUS ROG (5 шт.) по цене 120000.00
```

## 8. Система разрешений

Система различает три типа пользователей с разными правами доступа.

### Владелец компании (owner)

| Операция | Разрешение |
|----------|------------|
| Просмотр компаний | ✅ Да (своя компания) |
| Создание компании | ❌ Нет (может владеть только одной) |
| Редактирование компании | ✅ Да |
| Удаление компании | ❌ Только через суперпользователя |
| Все операции с Storage | ✅ Да |
| Все операции с Supplier | ✅ Да |
| Все операции с Product | ✅ Да |
| Все операции с Supply | ✅ Да |
| Все операции с Sale | ✅ Да (с ограничениями фильтра) |

### Сотрудник компании (employee)

| Операция | Разрешение |
|----------|------------|
| Просмотр компании | ✅ Да |
| Создание компании | ❌ Нет |
| Редактирование компании | ❌ Нет |
| Удаление компании | ❌ Нет |
| Просмотр Storage | ✅ Да |
| Удаление Storage | ❌ Нет |
| Просмотр Product | ✅ Да |
| Удаление Product | ❌ Нет |
| Операции с Supply | Зависит от конфигурации |
| Операции с Sale | Зависит от конфигурации |

### Новый пользователь (без компании)

| Операция | Разрешение |
|----------|------------|
| Создание компании | ✅ Да |
| Просмотр Storage | ❌ Нет (нет компании) |
| Просмотр других данных | ❌ Нет |

## 9. Полный рабочий процесс

Типичный сценарий работы с системой:

### 1. Регистрация и создание компании

```
1. Зарегистрируйтесь в системе
2. Войдите в админ-панель (/admin/)
3. Создайте компанию (COMPANY → Add Company)
```

### 2. Настройка инфраструктуры

```
4. Создайте склад (STORAGE → Add Storage)
5. Добавьте поставщиков (SUPPLIER → Add Supplier)
```

### 3. Работа с товарами

```
6. Создайте товары (PRODUCT → Add Product)
7. Сделайте поставку (SUPPLY → Add Supply)
8. Добавьте товары к поставке (SUPPLYPRODUCT → Add Supply product)
```

### 4. Продажи

```
9. Оформите продажу (SALE → Add Sale)
10. Добавьте товары к продаже (PRODUCTSALE → Add Product sale)
```

### Пример данных для тестирования

**Компания:**
- ИНН: `123456789012`
- Название: Test Company LLC

**Склад:**
- Адрес: `123 Main Street, City`

**Поставщик:**
- Название: Test Supplier
- Контакт: `supplier@example.com`

**Товар:**
- Название: Test Product
- Закупочная цена: 100.00
- Цена продажи: 150.00
- Количество: 50

## Важные замечания

1. **Авторизация обязательна** — без входа доступ к админке невозможен
2. **Фильтрация по компании** — списки объектов фильтруются по компании пользователя
3. **Ручное обновление остатков** — в админке можно изменять количество товаров вручную
4. **API vs Админка** — через API остатки обновляются автоматически, через админку — требуется ручное управление
5. **Ограничения SaleAdmin** — Sale использует CompanyFilterMixin, что может ограничивать видимость продаж
