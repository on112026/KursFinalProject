# Инструкция по настройке и запуску проекта CRMLite

## Предварительные требования
- Python 3.8 или выше
- Git (опционально, для клонирования репозитория)

## Шаг 1: Создание виртуального окружения
Создайте виртуальное окружение для изоляции зависимостей проекта:

```bash
git clone https://github.com/on112026/FinalProject02.git
cd FinalProject02
python3 -m venv venv
```

## Шаг 2: Активация виртуального окружения
Активируйте виртуальное окружение:

**Linux/Mac:**
```bash
source venv/bin/activate
```

**Windows:**
```bash
venv\Scripts\activate
```

## Шаг 3: Установка зависимостей
Установите все необходимые пакеты из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```

## Шаг 4: Настройка базы данных
Проект использует SQLite, база данных уже создана. Если нужно сбросить миграции:

```bash
python manage.py migrate
```

## Шаг 5: Создание суперпользователя (опционально)
Для доступа к админ-панели создайте суперпользователя:

```bash
python manage.py createsuperuser
```

## Шаг 6: Запуск сервера разработки
Запустите Django сервер:

```bash
python manage.py runserver
```

## Шаг 7: Доступ к API
- **API документация**: http://127.0.0.1:8000/
- **Админ-панель**: http://127.0.0.1:8000/admin/
- **API эндпоинты**: http://127.0.0.1:8000/api/

## Шаг 8: Использование API
1. **Регистрация пользователя**: POST `/api/register/`
2. **Вход в систему**: POST `/api/login/`
3. **Создание компании**: POST `/api/company/` (требуется аутентификация)
4. **Создание склада**: POST `/api/storage/` (требуется аутентификация)

## Шаг 9: Тестирование
Вы можете использовать инструменты вроде Postman или curl для тестирования API эндпоинтов.

## Важные примечания
- Проект работает в режиме DEBUG
- База данных хранится в файле `db.sqlite3`
- JWT токены действительны 60 минут
- Все API эндпоинты защищены JWT аутентификацией

## Устранение проблем
- Если возникают ошибки при установке зависимостей, обновите pip: `pip install --upgrade pip`
- Если сервер не запускается, проверьте порты: `netstat -tulpn | grep :8000`
- Для продакшена необходимо настроить SECRET_KEY и ALLOWED_HOSTS

## Завершение работы
Для деактивации виртуального окружения используйте команду:
```bash
deactivate
```

## Дополнительные команды
- **Создание миграций**: `python manage.py makemigrations`
- **Проверка кода**: `python manage.py check`
- **Запуск тестов**: `python manage.py test`
